Welcome — quick checklist

  1. Open a PeopleSoft Classic or Fluid page
  2. Confirm the MPU bar appears (Environment / Page Info / Inspect)
  3. Add a Favorite with ★ (optional Page Info template + notes)
  4. Optional: set Trace presets if you have UTILITIES security
  5. Optional: open the Side Panel from the extension popup

Tracing profiles

Flags applied when Trace is turned on from the bar. Requires access to delivered trace components.

When to use which flags: Start with SQL Statement + Bind when a page feels slow or returns wrong data — binds show the literal values sent to the database. Add PeopleCode Assignments or Start of programs when a field change, save, or row event looks wrong but SQL is clean. Use Verbose only for short, targeted sessions — it adds a lot of log noise. Turn Trace off when you are done.

Environments

Host allowlist (optional)

Default: MPU runs on any host matching PeopleSoft psp/psc URL patterns (equal access to classic PS Utilities). Enable this toggle to restrict to org origins only.

When enabled with an empty list, MPU will not inject on any page.

About

Morris PeopleSoft Utilities is maintained by hackmods (Ryan Morris).

Inspired by PS Utilities (Uffe Graakjaer, Neil Yetman, and contributors). This is a greenfield Manifest V3 rewrite for Chrome Web Store standards, PeopleSoft Fluid, and AODA / WCAG 2.1 AA.

  • No password storage
  • No telemetry or analytics
  • Settings stored only in chrome.storage.local
